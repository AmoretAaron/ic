The Artifact Manager stores artifacts to be used by this and other nodes in the same subnet in the artifact pool.

Required methods

fn on_artifact(
    &self,
    msg: Artifact,
    advert: GossipAdvert,
    peer_id: &NodeId
) -> Result<(), OnArtifactError<Artifact>>

When a new artifact is received by Gossip, it is forwarded to the ArtifactManager together with its advert via the on_artifact call. This then forwards them to be processed by the corresponding ArtifactClient/ArtifactProcessor based on the artifact type. Returns OnArtifactError if no clients were able to process it or an error has occurred.

See ArtifactClient::on_artifact for more details.

fn has_artifact(&self, message_id: &ArtifactId) -> bool

Checks if any of the ArtifactClient already has the artifact in the pool by the given identifier.

fn get_filter(&self) -> ArtifactFilter

Gets the filter that needs to be sent with re-transmission request to other peers. This filter should be a collection of all filters returned by the ArtifactClients.

See ArtifactClient::get_filter for more details.
