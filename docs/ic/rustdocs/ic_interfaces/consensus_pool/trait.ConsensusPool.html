The consensus pool contains all the artifacts received by P2P and produced by the local node.

It contains two sections:

  • The validated section contains artifacts that have been validated by consensus. To support resumability this section must be persistent.

  • The unvalidated section contains artifacts that have been received but haven't yet been validated. This section is in-memory only and thus volatile.
