fn register_client(
    &self,
    client_type: TransportClientType,
    async_event_handler: Arc<dyn AsyncTransportEventHandler>
) -> Result<(), TransportErrorCode>

Register the transport client of the specified type. No more than one module can register against a particular client type. This returns a handle to the client's context, which should be supplied in all the future interactions with the transport layer.

fn start_connections(
    &self,
    client_type: TransportClientType,
    peer: &NodeId,
    node_record: &NodeRecord,
    registry_version: RegistryVersion
) -> Result<(), TransportErrorCode>

Mark the peer as valid neighbor, and set up the transport layer to exchange messages with the peer. This call would create the necessary wiring in the transport layer for the peer:

    1. Set up the Tx/Rx queueing, based on TransportQueueConfig.
    1. If the peer is the server, initiate connection requests to the peer server ports.
    1. If the peer is the client, set up the connection state to accept connection requests from the peer.

fn stop_connections(
    &self,
    client_type: TransportClientType,
    peer_id: &NodeId,
    registry_version: RegistryVersion
) -> Result<(), TransportErrorCode>

Remove the peer from the set of valid neighbors, and tear down the queues and connections for the peer. Any messages in the Tx and Rx queues for the peer will be discarded.

fn send(
    &self,

) -> Result<(), TransportErrorCode>

Send the message to the specified peer. The message will be enqueued into the appropriate TxQ based on the TransportQueueConfig.
