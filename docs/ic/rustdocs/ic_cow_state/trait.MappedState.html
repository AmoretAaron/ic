fn soft_commit(&self, _pages: &[u64])

soft_commit ensures that all modified pages (mutations) represented by pages slice become part of the "current" state. This means even after MappedState object is dropped, mutations will be reflected in subsequent MappedState object and can eventually become part of some round. Mutations not soft_committed have a lifetime of MappedState object and will disappear once MappedState object is dropped.

fn update_heap_page(&self, _page_idx: u64, _bytes: &[u8])

Using update_heap_page, individual heap pages can be modified. This is useful during canister installation to patch up initial pages. Once the heap is modified, soft_commit is necessary to ensure that the modifications become part of "current" state.
