MAX_INTER_CANISTER_PAYLOAD_IN_BYTES

This sets the upper bound on how big a single inter-canister request or response can be. We know that allowing messages larger than around 2MB has various security and performance impacts on the network. More specifically, large messages can allow dishonest block makers to always manage to get their blocks notarized; and when the consensus protocol is configured for smaller messages, a large message in the network can cause the finalization rate to drop.

MAX_XNET_PAYLOAD_IN_BYTES

The maximum size of an inter-canister request or response that the IC can support.
